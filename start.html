About



Why?

This program was written for a school project as part of a course. The requirements were communicated to us gradually, and we tried our best until it worked. The project was structured according to the SCRUM principle to learn the system. For this project, we had half a year, where we had a Sprint Review every 3 weeks and a Daily Scrum meeting every week, as we only had 4 hours per week.


Who&How?

Our team consists of 3 students who tried to solve the project in synchronization with each other. In addition to programming, we also invested a lot of time in planning to achieve the best possible result. In the beginning, we brainstormed the user stories that we wanted to implement and assigned a value (story points) to them so that we knew the importance of features for the customer. After that, we selected various user stories that we wanted to implement in a sprint (3 weeks).


What?

The connection to the Mbot is facilitated through TCP/UDP, requiring an internet connection between the website and Mbot. Additionally, up to 3 MBots can be connected, and users can choose which one to control. The control can be done using a keyboard, controller (PS4, XBOX One), or even touch for mobile devices.


Account

To use the Mbot, you need to be logged in to save the data for future add-ons, features, and premium content for the future. Your data is stored in a Google Firebase database, which is protected and encrypted by provided custom security measures. If you press "Stay logged in," a cookie is stored on the website to enable this feature.
